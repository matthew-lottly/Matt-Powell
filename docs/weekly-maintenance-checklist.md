# Weekly Maintenance Checklist

1. Review open PRs and confirm whether backlog work should be merged, split, or closed.
2. Check `docs/portfolio-matrix.md` against the current local project state.
3. Update showcase priorities if a stronger project emerges.
4. Remove any generated files or tracked cache noise before commits.
5. Open or refresh standalone-repo issues for the highest-value next gaps.
6. Confirm README links in the hub still point to the intended public repositories.
7. Recheck whether any local planning docs have become stale or redundant.
