# Public Repo Roadmap

This roadmap keeps the public side of the account small, coherent, and easy to review.

## Public Strategy

- Keep 4 to 6 high-quality public repositories.
- Keep experiments, employer-adjacent work, and rough drafts private.
- Archive anything public that weakens the first impression.

## Recommended Repo Lanes

### 1. Flagship integrated system

Goal: show software engineering as the backbone while GIS, databases, and analytics reinforce it.

Good examples:

- Spatial data API with a backing database and analysis workflow
- Map-based application with ingestion, processing, and visualization
- Operational data platform with geospatial components

### 2. Backend and data systems

Goal: show architecture, data modeling, automation, and maintainable engineering.

Good examples:

- ETL pipeline
- PostGIS-backed service
- Database schema and migration workflow

### 3. GIS and geospatial application

Goal: show domain depth in spatial workflows and tools.

Good examples:

- Geoprocessing toolchain
- Spatial analysis workflow
- GIS automation utility

### 4. Analytics or data science project

Goal: show decision-making value, not just notebooks.

Good examples:

- Forecasting or classification project with clear outputs
- Reproducible analysis with documented assumptions
- Dashboard or reporting workflow

### 5. Optional developer tool

Goal: show code quality and reuse.

Good examples:

- CLI utility
- Python package
- Internal-quality tooling that can be generalized publicly

## Pinned Repo Order

1. Flagship integrated project
2. Backend or database project
3. GIS project
4. Analytics project
5. Optional developer tool

## Selection Rules

- Prefer one polished repo over three partial ones.
- Prefer applied projects over toy examples.
- Prefer repos with screenshots, diagrams, or sample outputs.
- Prefer repos with recent maintenance or a clear completed state.

## Public Readiness Checklist

- Clear repository name
- Clean README
- No secrets or internal company details
- Install and run steps work
- Example output or screenshots included
- License chosen if appropriate
- Topics added for discoverability
