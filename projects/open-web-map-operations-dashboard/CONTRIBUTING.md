# Contributing to open-web-map-operations-dashboard

## Scope

Use this repository for frontend map behavior, filter workflows, accessibility work, browser interaction tests, and reviewer-facing visual assets.

## Development Setup

```bash
npm install
npm test
npm run test:e2e
```

## Pull Request Expectations

1. Keep UI changes tied to a clear reviewer or operator workflow.
2. Update tests when filters, interaction flow, or responsive behavior changes.
3. Refresh screenshots when major UI changes land.
4. Avoid committing local build output or environment-specific files.

## Issue Reports

Include the screen, filter sequence, viewport size, and expected interaction path when reporting a bug or requesting a UI improvement.
